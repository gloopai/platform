# 上线变更说明（2026-03-24）

## 1. 变更目标

- 提升开放接口安全性与可观测性；
- 降低签名重放、登录爆破、接口流量冲击风险；
- 统一关键入口错误响应格式，便于商户接入与排障。

## 2. 本次上线内容

### 2.1 开放签名接口防重放

- 开放签名接口（如 `POST /v1/payin/order`、`POST /v1/payout/order`、`GET /v1/payin/query`、`GET /v1/payout/query`）新增必填签名字段：
  - `timestamp`（秒级 Unix 时间戳）
  - `nonce`（随机串）
- 服务端校验规则：
  - 时间窗校验（默认 `timestamp` 在当前时间 `+-300s` 内）
  - 重放校验（同 `merchant_id + nonce + timestamp` 仅允许一次）
- 重放命中返回：
  - HTTP `409`
  - `code = REPLAY_REQUEST`

### 2.2 登录与开放接口限流

- 新增 Redis 固定窗口限流（`INCR + EXPIRE`）：
  - 开放签名接口：按 `IP + merchant_id` 限流
  - 登录接口：按 `IP + account + path` 限流
- 超限返回：
  - HTTP `429`
  - `code = TOO_MANY_REQUESTS`

### 2.3 关键入口错误体统一

- 登录相关入口已统一为开放接口错误体：

```json
{
  "code": "SOME_CODE",
  "message": "human readable message"
}
```

### 2.4 关键资金链路结构化日志增强

- 在代付下单/扣款失败补偿/回调入账等阶段增加结构化日志字段：
  - `request_id`
  - `merchant_id`
  - `merchant_order_no`
  - `order_no`
  - `stage`
  - `err`（失败场景）

## 3. 配置变更

文件：`backend/services/gateway/etc/gateway-api.yaml`

新增：

- `ReplayGuard.*`
- `RateLimit.*`

示例（按环境修改）：

- `ReplayGuard.RedisAddr`
- `ReplayGuard.RedisPassword`
- `ReplayGuard.RedisDB`
- `ReplayGuard.AllowedSkewSeconds`
- `ReplayGuard.TTLSeconds`
- `RateLimit.RedisAddr`
- `RateLimit.RedisPassword`
- `RateLimit.RedisDB`
- `RateLimit.OpenAPILimitPerWindow`
- `RateLimit.OpenAPIWindowSeconds`
- `RateLimit.LoginLimitPerWindow`
- `RateLimit.LoginWindowSeconds`

## 4. 接口兼容性说明

- 商户侧签名请求必须新增并签名 `timestamp`、`nonce`；
- 若继续复用旧签名请求体，会命中防重放（`409 REPLAY_REQUEST`）；
- 高并发情况下可能出现限流（`429 TOO_MANY_REQUESTS`），应按窗口退避重试。
