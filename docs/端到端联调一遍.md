# 端到端联调一遍（主路径 MVP）

按顺序做完即可验证 **商户下单 → 收银台支付 → 模拟回调 → 商户侧可见**；不要求一次配齐 Consul/NSQ/Redis（缺省时脚本会提示，部分能力可能降级）。

---

## 0. 一次准备

| 项 | 说明 |
|----|------|
| MySQL | 本机 `3306` 可连；库表与演示数据：见 [`backend/deploy/init_demo.sh`](../backend/deploy/init_demo.sh)（默认库名 `pay`，密码与脚本内一致） |
| 依赖 | **推荐**：Consul `:8500`、Redis `:6379`、NSQ（脚本可自动起）；缺省时仅可能影响服务发现/通知，主链路可先试 |

```bash
# 在项目根目录（按你本机改 MYSQL_PASS）
bash backend/deploy/init_demo.sh
```

演示账号（与 `seed_demo.sql` 一致）：

- 管理台：`admin` / `admin123`
- 商户台：`m_demo` / `demo_secret`
- 开放下单：`merchant_id=m_demo`，`app_secret` 与库中 `api_secret` 一致（默认 `demo_secret`）

---

## 1. 启动后端与前端

```bash
bash dev-up.sh
```

脚本会拉起 trade / core / gateway（`:8080`）等，并在未占用时启动前端开发服务：

| 服务 | 默认地址 |
|------|----------|
| 网关 | http://127.0.0.1:8080/ |
| 收银台 | http://127.0.0.1:5174/ |
| 商户台 | http://127.0.0.1:5175/ |
| 管理台 | http://127.0.0.1:5176/ |

日志：项目根目录 `.dev-logs/*.log`。

---

## 2. 商户侧：创建订单

1. 打开 **商户台** → **开发配置**（`/developers`）。
2. 确认 `merchant_id`、`app_secret` 与演示商户一致。
3. **支付产品** 选 `mock`（或与 `pay_products` / 白名单一致的编码）。
4. 点击 **创建订单**，得到 `order_no` 与 `checkout_url`（指向本机收银台）。

可选：同一页 **查单联调** 用 `GET /v1/pay/query` 核对原始 JSON。

---

## 3. 收银台：发起支付

1. 点击 **打开 checkout_url**，或手动打开：  
   `http://127.0.0.1:5174/?order_no=<平台订单号>`
2. 选择支付方式（列表来自 `GET /v1/terminal/order` 的 `pay_products`）。
3. 点击 **确认支付** / **发起支付**：会调用 `POST /v1/terminal/pay`；PC 常出现 **扫码**弹窗（演示用二维码）。

---

## 4. 模拟上游回调（置成功）

在商户台 **开发配置** 下半部分 **模拟上游回调**：

- `channel_id`：创建订单返回里一般有（mock 路由后）；与 `seed_demo` 通道 id 一致即可。
- `channel_sign_secret`：演示通道与 `seed_demo.sql` 一致时为 `channel_secret`（或你改库后的值）。
- 点击 **触发支付成功**（调用 `POST /v1/callback/notify`）。

收银台若仍开着，待支付轮询或点 **刷新状态**，应变为 **支付成功**。

---

## 5. 商户侧：核对结果

- **交易管理**：订单状态、支付产品展示名与编码。
- **查单联调**：再看 `GET /v1/pay/query` 返回是否已为成功态。

（若配置了通知：商户 `notify_url` 可收到异步通知；MVP 不强制先验通知。）

---

## 6. 常见问题

| 现象 | 排查 |
|------|------|
| 网关 404 / 连不上 | 确认 `gateway` 进程与 `.dev-logs/gateway.log` |
| 创建订单签名校验失败 | `merchant_id` / `secret` 与库一致；本机 IP 是否在商户 `ip_whitelist`（演示为 `127.0.0.1`） |
| 收银台「查询订单失败」 | `order_no` 是否正确；网关与 `vite` 代理是否指向 `:8080` |
| 回调失败 | `channel_id`、`channel_sign_secret` 与通道表一致；见 `.dev-logs/gateway.log` |
