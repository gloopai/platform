## 2026-03-22：order 启动报错 `field "redis.Host" is not set`

**现象**

- `go run . -f etc/trade.yaml` 启动失败：

```
error: config file etc/trade.yaml, field "redis.Host" is not set
exit status 1
```

**原因**

- `trade` 的配置结构体内嵌了 `zrpc.RpcServerConf`，其中自带一个 `Redis` 字段（类型为 `RedisKeyConf`，必填 `Redis.Host` / `Redis.Key`）。
- 同时 `trade` 自己又定义了业务用 Redis 配置字段 `Redis`（使用 `Addr/Password/DB`），导致 YAML 里写了 `Redis.Addr`，但 go-zero 的配置校验在校验 `RpcServerConf.Redis` 时找不到 `Redis.Host`，从而直接退出。

**处理**

- 业务 Redis 配置字段从 `Redis` 改为 `BizRedis`：
  - [trade/config.go](file:///Users/evanqi/code/gloopai/pay/pay-platform/backend/services/trade/internal/config/config.go)
  - [trade/servicecontext.go](file:///Users/evanqi/code/gloopai/pay/pay-platform/backend/services/trade/internal/svc/servicecontext.go)
- `trade.yaml` 同时提供：
  - `Redis.Host` / `Redis.Key`（满足 go-zero `RpcServerConf.Redis` 的结构与校验）
  - `BizRedis.Addr/Password/DB`（满足业务使用的 go-redis client）
  - [trade.yaml](file:///Users/evanqi/code/gloopai/pay/pay-platform/backend/services/trade/etc/trade.yaml)

**验证**

- `CGO_ENABLED=0 go run . -f etc/trade.yaml` 正常启动并打印 `Starting rpc server at ...`

## 2026-03-22：服务已注册但 Consul UI 看不到/很快消失

**现象**

- 服务启动后 Consul UI 里看不到对应的 service，或短暂出现后消失。
- `/v1/health/service/<service>?passing=true` 查不到实例。

**原因**

- 本机 `8500` 端口被 `com.docke` 占用，Consul 运行在 Docker 容器内。
- 服务注册时健康检查使用 `127.0.0.1:<port>`，但对容器内的 Consul 来说，`127.0.0.1` 指向容器自身而非宿主机，TCP check 失败。
- 注册 payload 里设置了 `DeregisterCriticalServiceAfter: 1m`，检查持续失败后服务被自动摘除。

**处理**

- 仍然保持服务地址 `Address=127.0.0.1`（便于宿主机的调用方访问）。
- 当检测到 Consul 节点名形似 Docker 容器 ID（12 位 hex）且服务地址为 `127.0.0.1/localhost` 时，将健康检查目标改为 `host.docker.internal:<port>`，让容器内的 Consul 能检查到宿主机上的服务端口。
**验证**

- 启动后 `curl -s 'http://127.0.0.1:8500/v1/health/service/payment.rpc.trade?passing=true'` 能获取到实例，并在超过 1 分钟后仍保持 `passing`。
