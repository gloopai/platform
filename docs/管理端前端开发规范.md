# 管理端前端开发规范（`frontend/apps/admin`）

与 [`admin-platform.md`](./admin-platform.md) 配合使用，后续迭代请遵守本节约定。

---

## 1. 后台接口请求

- **统一入口**：所有需携带 **`X-Admin-Token`** 的接口必须经 `src/lib/adminApi.ts` 中的 **`adminRequest`**（或由其实现的 **`adminGet` / `adminPost` / `adminPut` / `adminDelete`**）发起。
- **禁止**：在业务页面或组件里手写 `fetch` 并重复拼鉴权头（除非该接口设计上不需要 Token，例如 **`POST /v1/admin/login`**，可单独 `fetch`）。
- **扩展**：若需额外 Header，使用 `adminRequest(path, { headers: { ... } })`，避免复制整段 `fetch` 样板代码。

---

## 2. 页面与模块目录

- **按业务模块建目录**：在 `src/views/modules/<模块名>/` 下放该模块的页面入口与子组件，例如 `modules/pay-products/`、`modules/channels/`、`modules/routing/`。
- **路由**：`src/router.ts` 仍集中注册路由；页面组件可指向 `views/modules/.../XxxPage.vue`。
- **拆分原则**：单文件只负责拼装与数据流（请求、状态、子组件协调）；展示块拆成子组件（列表、表单、表格、工具栏等），**类型**可放在同目录 `types.ts`。
- **遗留页面**：`views/pages/` 下尚未迁移的页面可在后续重构时按同样规则迁入 `views/modules/`，不必一次性改完。

---

## 3. 长列表与大量选项（产品 / 通道等）

- **侧栏列表**（如支付产品列表、通道列表）：提供 **搜索框**（按 ID、名称、类型等客户端过滤）；列表区域 **固定最大高度 + 内部滚动**（如 `max-h-[min(70vh,560px)]`），避免整页被撑得很长。
- **上百条通道的下拉选择**：不要用原生 `<select>` 渲染全部 option；使用 **可搜索 + 滚动面板**（如 `components/ChannelPicker.vue`），未输入时可 **限制首屏条数**、提示用户输入关键字缩小范围；**已绑定/需排除的 ID** 通过 props 传入过滤。
- **表格**：行数多时对表格外层加 **max-height + overflow-auto**，表头可用 `sticky` 便于对照。
