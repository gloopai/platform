核心定位
本系统定位为高性能、高可用的支付中台，向下对接各种支付渠道（银联、支付宝、微信、Stripe等），向上为商户提供统一的 API 接入与资金结算服务。

**重要架构说明（支付产品 vs 上游通道）**

对商户与终端用户展示的是 **支付产品**（如微信支付、支付宝）；平台内部可配置 **多条上游通道** 对应同一产品，并按成功率、商户、成本等自动或策略路由。**详见 [`通道与支付产品.md`](./通道与支付产品.md)**，实现与排期见 [`开发计划-分模块明细.md`](./开发计划-分模块明细.md)。

一、 系统功能全景图
1. 总管理台 (Platform Admin) —— “运营与分润大脑”
渠道路由中心 (Core)：

动态配置：图形化配置上游接口参数（网关、密钥、证书）。

智能调度：根据通道稳定性（成功率）、成本（费率）、响应速度自动切换。

熔断器：若某通道连续报错，自动下线并触发运维告警。

财务结算系统：

多级分润：自动计算平台服务费、代理商佣金、商户应结算金额。

对账模块：每日自动下载上游账单，与本地数据库比对（差错处理：补单、退款、挂账）。

风控引擎：基于 IP 频率、单笔金额限制、同卡高频交易拦截、地区黑名单。

代理商管理：支持开通多级代理，查看各级代理的交易量与分润统计。

2. 商户平台 (Merchant Hub) —— “资金与技术中枢”
自助入驻：商户资质上传、实名认证、在线签署协议。

支付资产管理：查看余额、D0/T1 结算记录、提现申请及进度。

开发者中心：

API Key / Secret 自助重置。

Webhooks 日志查询：可查看系统回调商户服务器的记录、返回码及重试次数。

集成交互式文档（Swagger/Redoc）。

交易分析：实时看板（今日流水、成功率百分比、客单价分析）。

3. 收银台 (Cashier UI) —— “转化率阵地”
智能适配：检测环境（微信、支付宝、移动浏览器、PC），自动隐藏不可用通道。

品牌化定制：商户可自定义 Logo、颜色主题，减少用户支付时的心理违和感。

极简流程：支持一键拉起 App 支付、扫码支付、H5 快捷支付。

4. 门户官网 (Official Website) —— “获客窗口”
产品介绍：聚合支付、代付业务、行业解决方案。

价格体系：费率阶梯、结算周期说明。

合规展示：展示相关安全认证、合作伙伴、版权信息。

二、 整合技术方案
1. 技术栈选型
核心后端：Go-zero (高性能微服务框架，自带流量治理、限流熔断)。

服务治理：Consul (注册中心、配置中心)。

异步通信：NSQ (轻量级、高可用消息队列，负责支付回调与异步落库)。

持久化层：MySQL (主库) + Redis (高频缓存、分布式锁、频率控制)。

前端生态：Vue3 + Tailwind CSS + Vite (极速构建，响应式 UI)。

2. 核心架构拓扑
API Gateway：统一入口，负责鉴权、签名校验、流量分发。

Order RPC：处理订单生命周期（创单、支付、过期、状态更新）。

Channel RPC：适配器模式设计，将不同上游的 API 抽象为统一标准接口。

Settle RPC：独立的清算服务，通过定时任务和消息队列确保账务 100% 正确。

三、 高并发与稳定性深度优化
1. 资金安全与一致性
幂等性保障：全局唯一 Order_No 结合 Redis 分布式锁，防止同一笔交易重复支付/加钱。

事务处理：结算加钱操作必须配合 Fund_Log (流水表) 在同一数据库事务内完成，确保存款与记录同步。

2. 高并发应对 (Performance)
读写分离：管理后台查询走从库，收银台创单走主库。

异步通知解耦：上游支付成功的回调先写入 NSQ，由消费者异步通知商户。即使商户服务器宕机，系统也会按阶梯时间（15s, 1m, 5m...）持续重试，不阻塞主流程。

3. 容灾与扩展
无状态设计：所有业务服务（Api/Rpc）均不存储本地状态，支持 K8s 快速水平扩容。

热备通道：通过管理台配置，实现秒级切换故障通道。

四、 后续演进建议
接入“代付”系统：除了收款，商户往往需要下发资金给其用户。

数据大屏：为总管理台增加可视化数据大屏，实时监控全国/全球交易分布。

SDK 封装：为商户提供原生 Android/iOS SDK，进一步提升支付体验。
