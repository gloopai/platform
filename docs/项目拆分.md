项目一：官方网站 (Official Portal)
定位：品牌展示、获客、开发者引流。

1. 首页 (Home)：

Banner区：聚合支付核心优势（稳定、高并发、多渠道）。

产品概览：快捷收单、代付业务、扫码支付、H5支付。

合作渠道：展示支付宝、微信、Stripe、银联等 Logo 墙。

2. 产品与服务 (Products)：

支付解决方案：电商、游戏、线下零售等行业案例。

代付/结算：介绍 T+0、D+0 极速结算能力。

3. 开发者中心 (Developer Docs)：

快速集成：API 调用流程图。

SDK 下载：提供 Go, PHP, Java, Python 示例代码压缩包链接。

在线文档：跳转至 Swagger 或自建的 GitBook 样式文档站。

4. 关于我们 & 合规 (About & Compliance)：

公司介绍、安全认证（PCI-DSS）、服务协议、隐私政策。

📱 项目二：收银台 (Checkout UI)
定位：极致简洁的支付转化页面，适配 H5/PC。

1. 订单信息区：

展示商户名称、订单金额（大字体）、订单编号、倒计时（防止订单过期）。

2. 支付方式选择区：

智能排序：根据用户环境（如微信内优先显示微信支付）。

列表展示：微信支付、支付宝、云闪付、网银、数字货币（可选）。

3. 交互组件：

扫码支付模态框：PC端自动弹出二维码，带“已支付”和“重新获取”按钮。

唤起 App 按钮：移动端点击直接 Schema 唤起支付 App。

4. 状态反馈页：

支付中：轮询动画。

支付成功/失败：明确的结果状态、错误码提示、自动跳转倒计时。

💼 项目三：商户平台 (Merchant Dashboard)
定位：商户自助财务与技术管理台。

1. 控制台 (Console)：

今日看板：今日流水、订单数、成功率、待结算余额。

2. 交易管理 (Transactions)：

订单列表：支持按时间、订单号、状态（成功/待支付/已退款）搜索。

详情弹窗：显示上游返回的原始信息、通知状态（已通知/通知失败）。

3. 财务中心 (Finance)：

结算记录：T+1/D+0 自动结算的流水详情。

提现申请：手动发起余额提现至绑定银行卡。

对账下载：支持按日下载 CSV/Excel 对账单。

4. 开发配置 (Developers)：

参数配置：商户 ID (mch_id)、AppSecret（支持重置）、IP 白名单。

回调设置：配置异步通知 URL (Notify URL)。

签名工具：前端模拟签名测试，验证自己写的代码是否正确。

🛠️ 项目四：总管理台 (Admin System)
定位：整个平台的“上帝模式”，侧重风控与调度。

1. 系统概览 (Global Stats)：

实时大屏：全平台 TPS、各通道成功率对比图、地域分布图。

2. 商户管理 (Merchant Management)：

商户入驻：审核资料、开通商户号。

费率配置：给不同商户设置不同的手续费（例如 A 商户 0.6%，B 商户 0.4%）。

黑白名单：封禁异常交易商户。

3. 通道/路由管理 (Channel & Routing)：

通道配置：添加上游（如：支付宝 H5 通道、微信扫码通道）。

智能路由：配置轮询权重、分流比例、根据金额段切换通道。

健康监控：实时查看上游接口延迟，一键全局禁掉某个报错严重的通道。

4. 运营与审计 (Audit & Ops)：

全站订单查询：跨商户搜索所有订单。

手动补单：对已经支付成功但商户未收到通知的订单进行“人工补发通知”。

管理员权限 (RBAC)：财务、技术、客服不同的操作权限。

💡 给 AI 的细化 Prompt 示例 (可以直接复制)
任务：请开发【总管理台】的“通道路由配置”模块。
技术栈：Vue3 + Tailwind CSS。
功能逻辑：

页面结构：左侧为通道列表，右侧为配置详情。

配置表单：需要包含通道名称、上游 API 地址、商户号、RSA 私钥文本域、权重（0-100）、单笔限额（最小值/最大值）。

高级功能：增加一个“熔断开关”，当该开关开启时，后端 RPC 需停止分流至此通道。

UI 要求：使用 Tailwind 的 Grid 布局，状态标签（运行中、已停用）要求颜色区分明显。
